1. how to solve CORS (Cross-Origin Resource Sharing) issues with webpack

    2. It is often the case when doing development in Webpack that you have your, e.g. React JXS code running in the webpack development server (the one you typically launch with npm run dev) and you need that front-end code to access via Ajax some back-end services, e.g. some JAX-RS module deployed in a Tomcat container.

    In those cases, one runs into CORS issues. To solve the problem you have to enable CORS on the backend module as described in this note.

    I have created an archetype project to exemplify the full chain, from a React front-end application running in webpack development server to a back-end JAX-RS module deployed in a Tomcat container.
