The following problem arises when one wishes to display a Dialog (<p:dialog>) from inside a form in order to select a value from a drop-down list or a secondary search screen (from inside the dialog). The button to display the Dialog is in many cases a p:commandLink with the "dlgName.show()" value set to the onclick attribute. This causes the problem that the parent (hosting) form is resubmitted since that is apparently the default behavior of the p:commandLink. The way to solve this problem is to set the onclick attribute equal to "dlgName.show();return false" as in the following, real-world example: 
    <p:outputLabel value="Οδός" for="customerStreet"/>
    <h:panelGrid columns="2">
        <p:inputText value="#{CustomerCEVController.customer.strId.name}" id="customerStreet" required="#{empty param['skipValidation']}"/>
        <p:commandLink type="button" onclick="Streetdlg.show(); return false" styleClass="ui-icon ui-icon-search">
        </p:commandLink>
    (...)
See also this SO post (where the advice given on using the appendToBody attribute, didn't work. Another alternative that also seems to be working is to use an h:outputLink instead of a p:commandLink as in the following real-world example: 
    <p:outputLabel for="customerMncName" value="Δήμος" />
    <h:panelGrid columns="2">
        <p:inputText value="#{CustomerCEVController.customer.mncId.name}" id="customerMncName" required="#{empty param['skipValidation']}"/>
        <h:outputLink onclick="Municipalitydlg.show(); return false" styleClass="ui-icon ui-icon-search">
        </h:outputLink>
