Get rid of the need to type-in passwords in SSH connections using SSH public key authorization

source

Assuming the following machines:

A
your local node
B
the node you wish to be able to hop to without having to enter a password every time
… do the following:
  1. on machine A:
    2. If the file ~/.ssh/id_rsa.pub does not already exist do a: 
    ssh-keygen -t rsa
    … and copy the long single line that this file contains into your clipboard
  2. on machine B
    3. Copy the line you placed into the clipboard at the end of file: ~/.ssh/authorized_keys (and make sure said file is readable)
  3. I have also encountered cases where it was necessary to do the following in machine B:
    • chmod chmod 700 .ssh
    • chmod 640 .ssh/authorized_keys
    … this is described in the source. Unless I applied those permissions ssh would still ask me for a password.
