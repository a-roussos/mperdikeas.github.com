1. Taking snapshots in a ZFS filesystem
    2. In a ZFS filesystem there is no need to use the custom script that rotates backups relying on a combination of cp -al and rsync. rsync alone is enough together with a ZFS pool, e.g. named 'backups' as in: 
    rsync -avx --delete --in-place /whatever /mnt/zfs
zfs snapshot backups@20120925
  2. Find the UUID of a drive in Ubuntu precise
    3. To find the drives: 
    sudo fdisk -l
    and then you do a: 
    sudo blkid /dev/sdb1
    where, in the above example, /dev/sdb1 is the device we're interested in.
  3. Deep compare two file hierarchies
    4. Either use meld or, from command line: 
    diff -qr a b | sort
  4. How to find mounted file systems and mount a new one
