1. procedure to automatically mount a new filesystem in Ubuntu
    1. get the UUID of the device in question:
      2. sudo blkid
    2. get the user and group of the id of the user who'll be the owner of the files on the mounted filesystem using the id command.
    3. add a line at the end of the /etc/fstab file. E.g.
      4. UUID=e6daad35-1f57-4281-8683-a03c20cfc5a3 /media/Elements ext4 defaults,uid=1000,gid=1000,umask=007
  2. Taking snapshots in a ZFS filesystem
    3. In a ZFS filesystem there is no need to use the custom script that rotates backups relying on a combination of cp -al and rsync. rsync alone is enough together with a ZFS pool, e.g. named 'backups' as in: 
    rsync -avx --delete --in-place /whatever /mnt/zfs
zfs snapshot backups@20120925
  3. Find the UUID of a drive in Ubuntu precise
    4. To find the drives: 
    sudo fdisk -l
    and then you do a: 
    sudo blkid /dev/sdb1
    where, in the above example, /dev/sdb1 is the device we're interested in.
  4. Deep compare two file hierarchies
    5. Either use meld or, from command line: 
    diff -qr a b | sort
  5. How to find mounted file systems and mount a new one
    6. The following history shows how to do it:
